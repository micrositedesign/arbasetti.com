Project Summary

The U.S. Department of Homeland Security website is designed for a broad and diverse audience. It has been shown that the target audience may find it difficult navigating the site. It is my intention to redesign the site to better accommodate its visitors.

My Responsibilities

I planned and developed the site redesign from start to finish. Responsibilities included user interviews, storyboarding, heuristic evaluations, wireframing and prototyping.

Proto-Persona

An educated guess of a potential DHS website user led to this proto-persona. Meet, Marlen! He is excited that the Pandemic restrictions are starting to ease up. He has been wanting to visit Japan for quite some time now, and he finds it necessary to know more about traveling outside of the United States.

As a user researcher, I want to understand what types of pains and gains people experienced using the Department of Homeland Security website.

Research Objective

To kickoff the project, it was found necessary to discover what type of research needed to be completed, and what type of information and uses someone like Marlen was hoping to obtain from a website like the U.S. Department of Homeland Security.

1. What did you first notice after entering the website?
2. Does anything stick out to you on this page?
3. What do you see or read that catches your eye?
4. How easy is the site to navigate?
5. Is anything difficult to understand?
6. Do you notice anything about the structure of the website?

Interview Plan

The newly created research objective led to the formation of an interview plan. Since time was a factor, a decision was made to conduct unstructured interviews with a few interrelated ideas to focus on when asking participants questions. The goal was to see what Marlen would see if he were the one being asked the questions and using the website.

User Interviews

Asking as many open-ended questions as possible while working on obtaining more in-depth information using probing questions based on responses. If something sounded like it could reveal something more important about the website, trying to follow up the participants answers with questions that would lead to further insights.

Affinity Diagram (Unsorted)

The interviews turned up some interesting ideas about the current state of the DHS website. Each interview idea was then written on a sticky note for further brainstorming and synthesizing.

Affinity Diagram (Sorted)

The sticky notes were then organized into natural relationships. This led to the processing, sorting, and identification of key insights regarding the data.

Card Sorting (Unsorted)

To help create an easily learnable and clear information architecture for the website, the site’s structure was written onto index cards. This facilitated how users may group information, data and features, and it provides insight into the mental model of users like Marlen.

Card Sorting Primary Navigation (Sorted)

The cards were then sorted to help identify and define the information architecture of the primary navigation of the potential site redesign.

Card Sorting Supplementary Navigation (Sorted)

The cards were then sorted to help identify and define the information architecture of the supplementary navigation of the potential site redesign.

Card Sorting Footer Navigation (Sorted)

The cards were then sorted to help identify and define the information architecture of the footer navigation of the potential site redesign.

Empathy Map

To understand the thoughts, feelings, and experiences of the DHS users, and by using the data from the interviews and affinity diagram, an empathy map was created to gain a deeper understanding of users like Marlen. His potential likes, dislikes, and pain points are all illustrated here.

User Persona

To create a realistic representation of DHS website user segments, a user persona was created based on qualitative and quantitative data gained from the previous user research.

Marlen needs to be able to easily accomplish real actions so he can travel outside the United States because the lack of site organization found on the U.S. Department of Homeland Security website makes it difficult to get things done.

User Insight Statement

To help clarify and develop a deep understanding of the user persona, a user insight statement was developed to tell the story of Marlen. This included his underlying motivations and unmet needs which could lead to potential solutions relating to the current issues facing the DHS website.

Part of the DHS website was designed to offer travel help for those traveling abroad. We have observed that the DHS website isn’t structured in a way that facilitates the actionable steps required to travel which may be causing visitors to find alternate ways of accomplishing traveling tasks. How might we improve the DHS website so that travelers can successfully travel abroad?

Problem Statement

A more concise description of the issue to be improved upon was developed to identify the gap between the way the current DHS website is now and the desired state of the website after the potential redesign.

I Like, I Wish, What If

To address user wants, things that add value for the user, emotional amplifiers, future technologies, new processes and to fix what is missing, the I Like, I Wish, What If... brainstorming technique was utilized to look at the way things are and what they could look like in the near or distant future.

Feature Prioritization Matrix

To identify the most important problems to solve first a feature prioritization matrix was created to evaluate the feasibility, desirability, and viability of the potential redesign.

Value Proposition Canvas

To help ensure the website redesign is positioned around the needs and values of users like Marlen, a value proposition canvas was instituted to describe the pain points and the potential benefits of a redesign.

Value Proposition Statement

The DHS website value proposition states the reason ‘why’ someone should use the DHS website. It can convince a user like Marlen why the website will be of more value to them than other companies who might offer similar services. This statement is why the Department of Homeland Security website is great, and it sets the website apart from others.

User Scenario

To show how a user might act to achieve a particular goal within the DHS website, a user scenario was created based on the data from the previous steps. This scenario defines Marlen as the user, Marlen’s goals, his actions, his feelings and the high-level funnel phases of the scenario.

Storyboard

This storyboard was created to visually predict and explore Marlen’s experience with the U.S. Department of Homeland Security website. This helps understand the flow of Marlen’s interaction with the website. It also gives an idea on how well a redesign would solve his problem, how it meets his criteria, and how it delivers a benefit to him.

User Journey Map

As a visual representation of Marlen’s experience with the DHS website, this user journey map analyzes the website from Marlen’s point of view. The map focuses on what is motivating Marlen when using the site. It addresses things like his needs, his hesitations, and his concerns. It also identifies areas of friction or pain points.

Wire Flow

These wire flows document a series of actions someone like Marlen would take to achieve a particular goal. This shows how users interact with the website as it currently exists. It also identifies the obstacles in the way of the user keeping them from achieving their goals.

Heuristic Evaluation

Based on Jakob Nielsen’s 10 General Principles for Interaction Design, this heuristic evaluation attempts to address specific Universal Design Principles concerning the U.S. Department of Homeland Security website.

Sitemap

This sitemap helps guide web structure and will sooner or later lead to the design of the actual user interface. This communicates the structure of seven branches of navigation that will be found on the redesigned DHS homepage.

Style Guide

Meant as a set of standards for developing the site redesign, this style guide ensures cohesiveness and implements a set of best practices regarding the usage of common user interface elements such as color and typography.

Wireframes

The wireframes created communicate the intention of how the site redesign will be laid out. It shows what elements will be on the page, and where they will be located, while focusing on content structure and interaction hierarchy. This wireframe will fundamentally evolve into a working prototype by connecting the information architecture to the visual design.

Prototypes

This prototype simulates how the finished website will work. This user experience/user interface deliverable will be used to test the website, and find ways to improve on it before the design is sent to production.
